Final Recommendations & Blueprint

Based on the analysis of constraints, data availability, and architecture efficiency, we propose three viable system designs. Click on a strategy to reveal the technical implementation details.

  • Architecture: Single Encoder-Decoder with two heads (RGB + Alpha/Depth).
  • Backbone: MobileViT or RepVGG blocks (Depthwise Separable Convs).
  • Training: Joint training on synthetic dataset (REDS + Depth Estimation).
  • Why: Shared feature extraction saves ~40% compute compared to running SR and Depth separately.
  • Architecture: Heavy SR model (IMDN/RFDN) + Simple Segmentation.
  • Bokeh Logic: Gaussian blur on background pixels (no true depth map).
  • Why: Depth estimation is expensive. If "portrait mode" style is enough, this allocates budget to sharper SR.
  • Architecture: Separate SR and Depth networks, both heavily pruned (INT8).
  • Technique: Train massive teachers for each task, then distill into MobileNets.
  • Why: Allows modular updates (improve SR without retraining Depth), but risks exceeding thermal limits.
